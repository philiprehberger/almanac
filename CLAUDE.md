# CLAUDE.md — Almanac

This file guides Claude Code when working in this repo.

## What this is

Almanac is the portfolio demo for an internal-docs RAG chatbot. Laravel 13 + Filament 5 for the chat API + admin; FastAPI Python worker for the embedding pipeline; Next.js 16 for the docs/demo surface; PostgreSQL 16 + pgvector ≥ 0.8 for chunk storage with **per-workspace partial HNSW indexes** and iterative-scan ACL pre-filtering.

The wedge against cheap "AI on our docs" demos: **citations that stream inline + ACL propagated from the source + a no-answer gap report + prompt-injection defenses + observable atomic re-index.** All five matter; missing any one collapses the pitch.

## What this isn't

- A production system. It's a portfolio demo. The README says "production-shaped, not production-grade" — match that framing.
- A "ChatGPT for your files" rebuild. Almanac is opinionated about citation rendering, ACL handling, and the no-answer signal; do not bury those opinions.
- Multi-model at v1. Embedder is locked to `text-embedding-3-small`; chat model is swappable via the adapter layer. Don't introduce per-workspace embedder choice in v1 code.

## Conventions

- **No Claude / AI attribution anywhere.** No `Co-Authored-By: Claude`, no "generated by …" in commits, code, comments, README, marketing copy. Project-wide.
- **Voice and tone match the proposal voice.** Diagnosis-first, em-dashes, concrete numbers, no hedging filler, no emoji, no exclamation points.
- **Mock-mode is the default for the portfolio demo.** `ALMANAC_CONNECTORS_MOCK_MODE=true` and `ALMANAC_DEFAULT_CHAT_PROVIDER=mock` keep the live demo working without real OAuth + LLM credentials. Real OAuth + real LLM keys go in a self-hosted deploy's `.env`; the same code paths run.
- **PHP 8.3 syntax** — readonly classes, enums with methods, named arguments, first-class callables.
- **Laravel 13.x** pinned.
- **PostgreSQL only.** No MySQL idioms — use `jsonb`, partial indexes, PG enums where appropriate. The `pgsql` connection is the default in `.env.example`.
- **ULIDs (`CHAR(26)`)** for primary keys on user-facing resources. `HasUlids` on the models.
- **RFC 7807 problem details** for API errors. Don't return `{"error": "..."}` shapes.
- **Bearer token auth** via `Authorization: Bearer alm_<prefix>_<secret>`. Hash at rest, store `prefix` + `last_four` plaintext for display.
- **OpenAPI-first.** Hand-curated `openapi/spec.yaml`, mirrored by route definitions; the `php artisan almanac:gen-openapi` command refreshes the source-of-truth file from route attributes.
- **No `// reason:` comments unless the WHY is non-obvious** — well-named identifiers describe what the code does.

## The load-bearing files

- `app/Services/Retrieval/AclAwareRetriever.php` — selectivity estimator + iterative-scan + two-pass fallback. The wedge collapses if this is wrong.
- `app/Services/Retrieval/PrincipalSetMaterializer.php` — resolves caller → principal set from `identity_mappings`. Fail-closed when an ACL names an unmapped principal.
- `app/Services/Llm/PromptBuilder.php` — wraps retrieved chunks in `<retrieved_chunk id="N" source="…">` tags and prepends the safety preamble.
- `app/Services/Llm/OutputFilter.php` — URL allowlist, image-tag rejection, hallucinated-citation rejection. Trips write to `prompt_injection_signals`.
- `app/Services/Llm/Adapters/*` — provider-specific structured-output enforcement. `MockProvider` is the fixture demo path.
- `app/Services/Chat/SseChatStreamer.php` — translates inline `<cite id="…"/>` tokens into `event: citation` frames preserving position.
- `database/migrations/2026_06_12_*_create_doc_chunks_table.php` — the vector column; embedder dim must match the adapter.
- `workers/embed/app.py` — FastAPI chunker + embedder + LLM-judge.

## Don't do

- Don't introduce a content-hash dedupe with ACL union. The plan explicitly cut it as a permission-bug factory; the UI shows a "duplicated content" badge instead.
- Don't post-filter ACL after a vanilla HNSW top-K — recall collapses for users with low selectivity. Use the iterative-scan path in `AclAwareRetriever`.
- Don't unwrap the `<retrieved_chunk>` tags in any prompt-construction code path. The tags are the safety boundary.
- Don't widen `prompt_edit` capability default — it gates the editor on the safety prompt template.
- Don't cache the api_key lookup. Revocation must be instant; the hashed-lookup happens uncached in middleware.
- Don't add a content-hash + ACL collapse path. Keep ACLs per-source.

## Mock-mode details

- `MockChatProvider` returns a canned answer that cites the top-K chunks the retriever passed in. It supports streaming via a simulated tokenizer.
- `MockEmbedProvider` produces deterministic 1536-dim vectors via a hash-based projection. Reproducible across runs; useful for tests.
- `MockDriveConnector` / `MockNotionConnector` / `MockSlackConnector` walk the fixture corpus under `database/seeders/fixtures/` instead of calling the real API.
- The fixture corpus includes two prompt-injection bait docs (one Drive, one Notion). The Phase 9 `PromptInjectionTest` exercises them.
